# 7. 요구사항 정의서

> **프로젝트명**: Synapse — 통합 학습-지식 그래프 SaaS
> **버전**: v1.0
> **작성일**: 2026-05-07
> **기술 스택**: Spring Boot 4, Flutter 3.x, FastAPI, PostgreSQL 16, Redis, Elasticsearch, Kafka, K8s

---

## 1. 개요

본 문서는 Synapse 시스템의 기능적 요구사항(FR)과 비기능적 요구사항(NFR)을 정의한다.

### ID 체계

- **FR**: `FR-{AREA}-{seq}` (AREA: AUTH, NOTE, CARD, SRS, GRAPH, AI, BILLING, EXPORT)
- **NFR**: `NFR-{AREA}-{seq}` (AREA: PERF, SEC, AVAIL, SCALE, UX)

### 우선순위 정의

| 등급 | 의미 |
|------|------|
| P0 | 필수 — MVP 출시 차단 요소 |
| P1 | 높음 — Phase 2 이전 구현 필요 |
| P2 | 중간 — Phase 3~4 구현 |
