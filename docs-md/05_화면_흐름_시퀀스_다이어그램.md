## 5.1 회원가입 + Tenant 생성

```mermaid
sequenceDiagram

    participant GW as Gateway
    participant AUTH as Auth Service
    participant PG as PostgreSQL
    participant RD as Redis
    participant KF as Kafka

    U->>GW: POST /auth/signup<br/>{email, password, displayName}
    GW->>AUTH: Forward (Rate Limit 통과)

    AUTH->>AUTH: 비밀번호 유효성 검증

    AUTH->>RD: SETEX refresh_token:{tokenHash} {7d} {userId}
    AUTH->>KF: Publish user.registered

    AUTH-->>GW: 201 Created + Set-Cookie (refresh_token)
    GW-->>U: {userId, tenantId, accessToken}

    Note over U: 대시보드로 리다이렉트
```

## 5.2 OAuth 로그인 + MFA 검증

```mermaid
sequenceDiagram
    participant U as 사용자 (Flutter)
    participant GW as Gateway
    participant AUTH as Auth Service
    participant GOOGLE as Google OAuth
    participant PG as PostgreSQL
    participant RD as Redis

    U->>GW: POST /auth/oauth/google
    GW->>AUTH: Forward
    AUTH-->>U: {redirectUrl: "https://accounts.google.com/..."}

    U->>GOOGLE: 브라우저 리다이렉트 (동의 화면)
    GOOGLE-->>U: Authorization Code + Redirect

    U->>GW: POST /auth/oauth/google/callback<br/>{code, state}
    GW->>AUTH: Forward
    AUTH->>GOOGLE: Exchange code → access_token
    GOOGLE-->>AUTH: {access_token, id_token}
    AUTH->>AUTH: id_token 디코딩 → email, sub

    AUTH->>PG: SELECT oauth_identities WHERE provider='google' AND provider_user_id=sub
    alt 기존 사용자
        PG-->>AUTH: user_id 반환
    else 신규 사용자
        AUTH->>PG: INSERT users + tenants + tenant_members
        PG-->>AUTH: 신규 user_id
    end

    AUTH->>PG: SELECT mfa_credentials WHERE user_id AND is_active=true
    alt MFA 설정됨
        AUTH-->>U: 200 {mfaRequired: true, mfaToken: "temp-jwt"}
        U->>U: TOTP 코드 입력 화면

        U->>GW: POST /auth/mfa/verify<br/>{mfaToken, code: "123456"}
        GW->>AUTH: Forward
        AUTH->>AUTH: TOTP 검증 (30초 윈도우)
        AUTH->>RD: SETEX refresh_token
        AUTH-->>U: 200 {accessToken, user}
    else MFA 미설정
        AUTH->>RD: SETEX refresh_token
        AUTH-->>U: 200 {accessToken, user}
    end

    Note over U: 대시보드 진입
```

## 5.4 AI 카드 자동 생성 (LLM + Cache)

```mermaid
sequenceDiagram
    participant U as 사용자
    participant GW as Gateway
    participant AI as AI Service
    participant RD as Redis
    participant PG as PostgreSQL
    participant LLM as OpenAI API
    participant CARD as Card Service

    U->>GW: POST /ai/cards/generate<br/>{noteId, cardType: "basic", count: 5}
    GW->>AI: + X-Tenant-Id

    AI->>PG: SELECT content_md FROM notes WHERE id=? AND tenant_id=?
    PG-->>AI: 노트 본문

    AI->>AI: 요청 해시 생성 (SHA-256)
    AI->>PG: SELECT FROM semantic_cache WHERE query_hash=? AND expires_at > now()

    alt 캐시 HIT
        PG-->>AI: 캐시된 카드 결과
        AI->>PG: UPDATE semantic_cache SET hit_count = hit_count + 1
    else 캐시 MISS
        AI->>RD: SETNX rate_limit:ai:{tenantId}:{month} (할당량 확인)
        RD-->>AI: 잔여 횟수

        alt 할당량 초과
            AI-->>GW: 403 {code: "AI_QUOTA_EXCEEDED"}
            GW-->>U: 할당량 초과 안내
        else 할당량 내
            AI->>AI: 프롬프트 구성 (시스템 + 노트 내용 + 지시사항)
            AI->>LLM: ChatCompletion API 호출<br/>model: gpt-4o, temperature: 0.7
            LLM-->>AI: 생성된 카드 JSON

            AI->>AI: 응답 파싱 + 유효성 검증
            AI->>PG: INSERT INTO semantic_cache (hash, result, expires_at=24h)
            AI->>PG: INSERT INTO llm_usage_logs (tokens, cost, latency)
            AI->>RD: INCR rate_limit:ai:{tenantId}:{month}
        end
    end

    AI-->>GW: 200 {cards: [...], usage: {...}}
    GW-->>U: 생성된 카드 미리보기

    U->>U: 카드 검토 + 선택
    U->>GW: POST /cards/batch<br/>{deckId, cards: [선택된 카드들]}
    GW->>CARD: Forward
    CARD->>PG: INSERT INTO cards (배치)
    CARD->>PG: UPDATE card_decks SET card_count += N
    CARD-->>U: 201 카드 저장 완료

    Note over U: 덱에서 새 카드 확인 가능
```

## 5.5 복습 세션 (SRS 스케줄링)

```mermaid
sequenceDiagram
    participant U as 사용자
    participant GW as Gateway
    participant CARD as Card Service
    participant PG as PostgreSQL
    participant RD as Redis
    participant KF as Kafka

    U->>GW: GET /reviews/queue
    GW->>CARD: + tenant_id, user_id

    CARD->>PG: SELECT cards WHERE tenant_id=? AND user_id=?<br/>AND status IN ('review','learning') AND due_date <= now()<br/>ORDER BY due_date ASC LIMIT 50
    PG-->>CARD: 오늘 복습할 카드 목록

    CARD->>PG: SELECT cards WHERE status='new' AND deck_id=?<br/>LIMIT (new_cards_per_day - today_new_count)
    PG-->>CARD: 오늘의 새 카드

    CARD-->>U: {totalDue: 25, cards: [...]}

    U->>GW: POST /reviews/sessions<br/>{deckId}
    GW->>CARD: Forward
    CARD->>PG: INSERT INTO review_sessions (status='in_progress')
    CARD-->>U: {sessionId}

    loop 각 카드 복습
        U->>U: 카드 앞면 표시
        U->>U: 생각 → 뒷면 확인
        U->>U: 난이도 선택 (Again/Hard/Good/Easy)

        U->>GW: POST /reviews/sessions/{sessionId}/submit<br/>{cardId, rating: 3, timeSpentMs: 4500}
        GW->>CARD: Forward

        CARD->>CARD: SM-2 알고리즘 계산
        Note right of CARD: rating=3(Good)일 때:<br/>new_EF = EF + (0.1 - (5-3)*(0.08+(5-3)*0.02))<br/>new_interval = old_interval * new_EF<br/>repetitions += 1

        CARD->>PG: UPDATE cards SET<br/>easiness_factor=new_EF,<br/>interval_days=new_interval,<br/>due_date=now()+new_interval,<br/>repetitions=repetitions+1,<br/>last_reviewed_at=now()
        CARD->>PG: INSERT INTO card_reviews (rating, time_spent, prev_ef, new_ef, ...)
        CARD->>PG: UPDATE review_sessions SET completed_cards += 1

        CARD-->>U: {nextInterval: 7, newEF: 2.6, progress: {completed: 10, total: 25}}
    end

    U->>GW: PUT /reviews/sessions/{sessionId}/complete
    GW->>CARD: Forward
    CARD->>PG: UPDATE review_sessions SET status='completed', ended_at=now()
    CARD->>KF: Publish review.session.completed
    CARD-->>U: {summary: {total: 25, correct: 20, accuracy: 0.8, totalTime: "12m"}}

    Note over U: 세션 결과 요약 화면 표시
```

## 5.6 시맨틱 검색 (Hybrid Retrieval)

```mermaid
sequenceDiagram
    participant U as 사용자
    participant GW as Gateway
    participant AI as AI Service
    participant PG as PostgreSQL
    participant ES as Elasticsearch
    participant RD as Redis

    U->>GW: POST /ai/search/hybrid<br/>{query: "과적합을 방지하는 정규화 기법", limit: 10}
    GW->>AI: + tenant_id

    AI->>RD: GET search_cache:{tenant_id}:{query_hash}
    alt 캐시 HIT (TTL 5분)
        RD-->>AI: 캐시된 결과
        AI-->>U: 즉시 반환
    else 캐시 MISS
        par 병렬 검색
            AI->>PG: 벡터 검색 (pgvector)<br/>SELECT note_id, chunk_text,<br/>1 - (embedding <=> query_vec) AS score<br/>FROM note_chunks WHERE tenant_id=?<br/>ORDER BY embedding <=> query_vec<br/>LIMIT 20
            PG-->>AI: 시맨틱 결과 (상위 20개)

            AI->>ES: BM25 검색 (Elasticsearch)<br/>GET /notes/_search<br/>{query: {match: {content: "과적합 정규화"}},<br/>filter: {term: {tenant_id: "..."}}}
            ES-->>AI: 키워드 결과 (상위 20개)
        end

        AI->>AI: RRF (Reciprocal Rank Fusion) 결합
        Note right of AI: RRF Score = Σ 1/(k + rank_i)<br/>k = 60 (상수)<br/>시맨틱 rank + BM25 rank 결합

        AI->>AI: 상위 10개 선택 + 스니펫 생성
        AI->>RD: SETEX search_cache:{hash} 300 {results}
        AI-->>GW: {results: [...]}
        GW-->>U: 검색 결과 표시
    end

    Note over U: 결과 클릭 시 노트 상세 화면으로 이동
```

## 5.7 Stripe 결제 (Checkout + Webhook)

```mermaid
sequenceDiagram

    participant GW as Gateway
    participant BILL as Billing Service
    participant STRIPE as Stripe API
    participant PG as PostgreSQL
    participant KF as Kafka

    GW->>BILL: + tenant_id

    BILL->>PG: SELECT stripe_customer_id FROM tenants WHERE id=?

        BILL->>STRIPE: POST /v1/customers {email, name, metadata}
        STRIPE-->>BILL: {customerId}
        BILL->>PG: UPDATE tenants SET stripe_customer_id=?
    end

    BILL->>STRIPE: POST /v1/checkout/sessions<br/>{customer, price_id, mode: "subscription",<br/>success_url, cancel_url}
    STRIPE-->>BILL: {sessionId, url}

    BILL-->>U: {checkoutUrl: "https://checkout.stripe.com/..."}
    U->>STRIPE: 브라우저 리다이렉트 (결제 정보 입력)
    STRIPE-->>U: 결제 완료 → success_url 리다이렉트

    Note over STRIPE: Stripe Webhook 비동기 발송

    STRIPE->>GW: POST /billing/webhooks<br/>(Stripe-Signature 헤더)
    GW->>BILL: Forward (서명 검증)

    BILL->>BILL: Webhook 서명 검증 (whsec_...)
    BILL->>BILL: 이벤트 타입 분기

    alt checkout.session.completed
        BILL->>PG: INSERT INTO subscriptions<br/>(tenant_id, plan_code='pro', stripe_subscription_id, status='active')
        BILL->>PG: UPDATE tenants SET plan='pro'
        BILL->>PG: UPDATE usage_counters SET max_value = (Pro 한도)
    else invoice.payment_failed
        BILL->>PG: UPDATE subscriptions SET status='past_due'
        BILL->>KF: Publish billing.payment.failed (알림용)
    else customer.subscription.deleted
        BILL->>PG: UPDATE subscriptions SET status='canceled'
        BILL->>PG: UPDATE tenants SET plan='free'
        BILL->>PG: UPDATE usage_counters SET max_value = (Free 한도)
    end

    BILL->>KF: Publish billing.subscription.changed
    BILL-->>STRIPE: 200 OK (Webhook ACK)

    Note over U: 다음 방문 시 Pro 기능 활성화 확인
```

```mermaid
    S3-->>WORKER: 업로드 완료

    WORKER->>PG: UPDATE data_export_jobs SET<br/>status='completed',<br/>download_url=presigned_url,<br/>expires_at=now()+24h,<br/>completed_at=now()

    WORKER->>KF: Publish gdpr.export.completed {jobId, userId}

    Note over U: 이메일 알림 또는 폴링으로 확인

    U->>GW: GET /me/data-export/{jobId}
    GW->>AUTH: Forward
    AUTH->>PG: SELECT FROM data_export_jobs WHERE id=?
    AUTH-->>U: {status: "completed", downloadUrl: "https://s3...", expiresAt: "..."}

    U->>S3: GET (Presigned URL)
    S3-->>U: ZIP 파일 다운로드
```

## 5.12 알림 발송 (복습 리마인더)

```mermaid
sequenceDiagram

    participant KF as Kafka
    participant NOTI as Notification Service
    participant PG as PostgreSQL
    participant RD as Redis
    participant FCM as FCM/APNs

    Note over CARD: 매일 오전 9시 (사용자 타임존) Cron
    CARD->>PG: SELECT user_id, COUNT(*) as due_count<br/>FROM cards WHERE due_date <= today<br/>GROUP BY user_id HAVING COUNT(*) > 0
    CARD->>KF: Publish card.review.due (각 사용자별)

    KF->>NOTI: Consume card.review.due

        alt 토큰 존재
            NOTI->>FCM: POST (푸시 발송)
```
