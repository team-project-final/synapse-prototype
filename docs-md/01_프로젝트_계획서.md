> **프로젝트명**: Synapse — 통합 학습-지식 그래프 SaaS
> **버전**: v1.0
> **작성일**: 2026-05-07
> **기술 스택**: Spring Boot 4, Flutter 3.x, FastAPI, PostgreSQL 16, Redis, Elasticsearch, Kafka, K8s

---

## 1.1 배경 및 문제 정의

### 현재 시장의 문제점

| 구분 | PKM 도구 (Obsidian, Notion) | SRS 도구 (Anki, Quizlet) |
|------|---------------------------|--------------------------|
| 강점 | 자유로운 노트 작성, 백링크 | 과학적 반복 학습 알고리즘 |
| 약점 | 복습 메커니즘 부재 | 맥락 없는 단편 카드 |
| 통합 | 수동 복사/붙여넣기 필요 | 원본 노트와 단절 |

### 핵심 문제

1. **PKM과 SRS의 분리**: 지식을 기록하는 도구와 학습하는 도구가 별개로 존재하여 워크플로우가 단절됨
2. **맥락 상실**: Anki 카드는 원본 노트와 연결되지 않아 깊이 있는 이해가 어려움
3. **수동 카드 생성 부담**: 노트에서 카드를 수동으로 만드는 작업이 학습 의욕을 저하
4. **지식 관계 시각화 부재**: 개념 간 연결 관계를 파악하기 어려움
5. **시맨틱 검색 부재**: 키워드 기반 검색만 가능하여 의미 기반 탐색 불가

## 1.3 대상 사용자

### 1.3.1 학습자 (Learner) — Primary

- **페르소나**: 대학생, 자격증 준비생, 평생학습자
- **니즈**: 효율적 복습, 지식 정리, 시간 절약
- **사용 패턴**: 일 30분~2시간, 모바일 복습 + 데스크톱 노트 작성
- **Pain Point**: 여러 도구 간 전환, 카드 수동 생성 부담

### 1.3.2 교육자 (Educator) — Phase 4+

- **페르소나**: 온라인 강사, 학원 교사, 기업 교육 담당자
- **니즈**: 학습 콘텐츠 배포, 학습자 진도 추적
- **사용 패턴**: 주 1~2회 콘텐츠 제작, 일 1회 대시보드 확인
- **Pain Point**: 학습 성과 측정 어려움

### 1.3.3 관리자 (Admin/운영)

- **페르소나**: Synapse 내부 운영팀
- **니즈**: 테넌트 관리, 시스템 모니터링, 사용자 지원
- **사용 패턴**: 상시 모니터링
- **Pain Point**: 멀티테넌시 관리 복잡성

### 1.3.4 스터디 그룹 멤버 (Study Group Member) — Phase 1

- **페르소나**: 함께 공부하는 학습 커뮤니티 참여자
- **니즈**: 학습 자료 공유, 그룹원과 동기부여, 공유 덱 활용
- **사용 패턴**: 주 3~5회, 그룹 활동 확인 + 공유 덱 복사 + 리더보드 비교
- **Pain Point**: 혼자 공부하면 지속력이 떨어짐, 좋은 카드를 찾기 어려움

## 1.4 프로젝트 범위

### 포함 (In Scope) — MVP Phase 1 (4주)

| 기능 | 설명 |
|------|------|
| 노트 CRUD | Markdown 에디터, 위키링크(`[[]]`), 태그 |
| 백링크/그래프 | 양방향 링크 자동 감지, 2D 그래프 시각화 |
| 카드/덱 관리 | 수동 카드 생성, 덱 분류 |
| SRS 복습 | SM-2 알고리즘 기반 스케줄링, 스와이프 UI |
| AI 카드 생성 | GPT-4o 기반 노트→카드 자동 변환 |
| 시맨틱 검색 | pgvector 임베딩 + 키워드 하이브리드 |
| 인증 | OAuth 2.0 (Google/GitHub) + JWT |
| 빌링 | Stripe 연동, Free/Pro 플랜 |
| 멀티테넌시 | Pool 모델 + RLS |
| Cmd+K 커맨드 팔레트 | 키보드 우선 UX, 전 기능 접근 |
| 온보딩 체크리스트 | 3단계 (첫 노트→첫 카드→첫 복습), TTFV 5분 |
| 복습 완료 축하 | 파티클 애니메이션, 학습 동기 부여 |
| 코드 구문강조 | 마크다운 코드 블록 하이라이팅 |
| 디자인 시스템 | DESIGN.md 기반 Warm Intellectual 미학 |
| 스터디 그룹 | 그룹 생성/가입/멤버 관리, 공개/승인/초대 방식 |
| 덱/노트 공유 | 공개/그룹/링크 공유, 복사, 평가 |
| 게이미피케이션 | XP/레벨/배지/스트릭/리더보드 |
| 알림 시스템 | 인앱/푸시(FCM/APNs), 복습 리마인더, 커뮤니티 알림 |
| 콘텐츠 모더레이션 | 신고 접수/처리, 콘텐츠 관리 |
| 관리자 확장 | 신고관리, 모더레이션, 그룹관리, 게이미피케이션 설정 |

### 제외 (Out of Scope)

| 기능 | 사유 | 예상 시점 |
|------|------|-----------|
| Enterprise 플랜 | B2B 영업 체계 미구축 | Phase 5+ |
| 데스크톱 앱 (Electron) | Flutter Web으로 충분 | 미정 |
| 교육자 대시보드 | Phase 4 로드맵 | 6개월+ |
| 오프라인 동기화 | 기술 복잡도 높음 | Phase 3 |
| 자체 LLM 학습 | 비용 대비 효과 불분명 | 미정 |
