## D.2 핵심 원칙

> **A 모듈이 B 모듈을 호출할 때, A 내부에 B에 대한 어댑터를 두고 도메인은 어댑터의 추상(Port)에만 의존한다.**

### D.2.1 5가지 규칙

1. **Outbound Port는 도메인 레이어에 둔다** — 인터페이스 형태, 외부 기술(gRPC/REST/Kafka)에 무지
2. **Adapter는 인프라 레이어에 둔다** — Port를 구현, 통신 프로토콜과 외부 SDK 캡슐화
3. **DTO ↔ Domain 변환은 어댑터 내부에서** — 외부 모델이 도메인으로 흘러들어오지 못하게
4. **Resilience4j / 로깅 / 트레이싱은 어댑터에 적용** — 도메인은 횡단 관심사로부터 격리
5. **테스트는 Port 인터페이스에 대해 작성** — 도메인 테스트는 Fake/Stub Adapter로

### D.2.2 왜 이렇게 하는가

| 효과 | 설명 |
|------|------|
| **Anti-Corruption Layer** | B의 외부 모델 변화(필드 추가/이름 변경)가 A의 도메인까지 번지지 않음 |
| **프로토콜 교체 자유도** | gRPC → REST → Kafka 전환 시 Adapter만 수정 |
| **테스트 용이성** | 도메인 테스트는 외부 의존성 없이 Fake Port로 가능 |
| **장애 격리** | Adapter에 Circuit Breaker / Fallback 적용 → 도메인은 정상 흐름만 처리 |
| **외부 SDK 잠금 회피** | Stripe → Toss, OpenAI → Anthropic 교체 시 Adapter만 추가 |

### D.3.1 의존성 방향 규칙

```
infrastructure.adapter → domain.port (구현)
infrastructure.adapter → application.usecase (호출 — inbound만)
application.usecase    → domain.port (호출)
domain.port            → (외부 의존성 없음)
domain.model           → (외부 의존성 없음)
```

**금지**: `domain` 패키지가 `infrastructure`나 외부 SDK(`io.grpc`, `com.stripe`, `org.springframework.kafka`)를 import.

ArchUnit으로 강제:

```java
// application/usecase/GenerateCardFromNoteUseCase.java
@UseCase
@RequiredArgsConstructor
public class GenerateCardFromNoteUseCase {
    
    private final NotePort notePort;                        // 인터페이스에만 의존
    private final CardRepository cardRepository;
    private final AICardGeneratorPort aiPort;
    
    @Transactional
    public Card execute(GenerateCommand cmd) {
        NoteSnapshot note = notePort.findForLearning(cmd.tenantId(), cmd.noteId())
            .orElseThrow(() -> new NoteNotFoundException(cmd.noteId()));
        
        // 도메인 로직: Note의 외부 표현(proto, JSON)은 전혀 모름
        List<CardCandidate> candidates = aiPort.generateFromNote(note, cmd.options());
        Card card = Card.fromCandidate(candidates.getFirst(), cmd.userId());
        return cardRepository.save(card);
    }
}
```

```java
// infrastructure/adapter/outbound/note/NoteGrpcAdapter.java
@Component
@RequiredArgsConstructor
@Slf4j
public class NoteGrpcAdapter implements NotePort {
    
    @GrpcClient("note-service")
    private final NoteServiceGrpc.NoteServiceBlockingStub stub;
    private final NoteMapper mapper;
    
    @Override
    @CircuitBreaker(name = "note-service", fallbackMethod = "fallback")
    @Retry(name = "idempotent-grpc")
    @TimeLimiter(name = "note-service")
    public Optional<NoteSnapshot> findForLearning(UUID tenantId, UUID noteId) {
        try {
            GetForLearningResponse response = stub
                .withDeadlineAfter(500, TimeUnit.MILLISECONDS)
                .getForLearning(GetForLearningRequest.newBuilder()
                    .setTenantId(tenantId.toString())
                    .setNoteId(noteId.toString())
                    .build());
            return Optional.of(mapper.toDomain(response));
        } catch (StatusRuntimeException e) {
            if (e.getStatus().getCode() == Status.Code.NOT_FOUND) {
                return Optional.empty();
            }
            throw new ExternalServiceException("note-service", e);
        }
    }
    
    @SuppressWarnings("unused")
    private Optional<NoteSnapshot> fallback(UUID tenantId, UUID noteId, Throwable t) {
        log.warn("note-service fallback: noteId={}, reason={}", noteId, t.getMessage());
        return Optional.empty();
    }
}
```

// infrastructure/adapter/outbound/kafka/CardEventKafkaAdapter.java
@Component
@RequiredArgsConstructor
public class CardEventKafkaAdapter implements CardEventPublisher {
    
    private final OutboxRepository outboxRepo;
    private final CloudEventFactory cloudEventFactory;
    private final TraceContextProvider traceProvider;
    
    @Override
    public void publishReviewed(Card card, ReviewRating rating, ReviewResult result) {
        var payload = CardReviewedPayloadMapper.from(card, rating, result);
        var event = cloudEventFactory.builder()
            .type("card.reviewed")
            .source("synapse/card-service")
            .subject("cards/" + card.getId())
            .tenantId(card.getTenantId())
            .userId(card.getUserId())
            .traceContext(traceProvider.current())
            .payload(payload)
            .build();
        
        // Outbox 패턴 (03-A.6) — Kafka 직접 발행 안 함
        outboxRepo.save(OutboxRecord.from(event, "card.reviewed"));
    }
}

# infrastructure/adapter/outbound/openai_adapter.py
from openai import AsyncOpenAI
from domain.port.llm_port import LLMPort

class OpenAIAdapter(LLMPort):
    
    def __init__(self, client: AsyncOpenAI, mapper: OpenAIMapper):
        self._client = client
        self._mapper = mapper
    
    @circuit_breaker(name="openai-api")
    @retry(stop=stop_after_attempt(3), wait=wait_exponential())
    async def generate_cards(self, request: GenerationRequest) -> list[GeneratedCard]:
        try:
            response = await self._client.chat.completions.create(
                model=request.model.value,
                messages=[...],
                response_format={"type": "json_object"},
                timeout=15.0,
            )
            return self._mapper.to_domain(response, request)
        except APIError as e:
            raise LLMExternalError("openai", e) from e

## D.5 안티패턴

| ❌ 안티패턴 | ✅ 올바른 방식 |
|------------|--------------|
| UseCase가 `NoteServiceBlockingStub` 직접 호출 | UseCase는 `NotePort`만 호출, Adapter가 stub 사용 |
| 도메인 모델에 proto 필드 (`getNoteId()`) 노출 | 어댑터에서 `UUID`로 변환해서 도메인 전달 |
| Port가 `import io.grpc.StatusRuntimeException` | Port는 도메인 예외만 throw, Adapter가 변환 |
| Adapter 내부에 비즈니스 로직 (`if (note.tag == "important") ...`) | Adapter는 변환만, 비즈니스는 도메인/UseCase |
| 한 Adapter가 여러 외부 svc 호출 (`NoteAndCardAdapter`) | Adapter는 외부 svc당 1개 |
| Port 인터페이스에 `KafkaTemplate` 시그니처 노출 | Port는 도메인 명령(`publishReviewed(Card, Rating)`)만 받음 |
| 외부 응답 DTO를 Controller까지 그대로 반환 | Mapper로 DTO → Domain → ResponseDto 변환 |
| Fallback에서 도메인 예외 발생 | Fallback은 안전한 기본값 또는 부분 응답 |
| 동기 호출이 필요 없는데 gRPC 사용 | Kafka 이벤트 어댑터로 대체 |

public class FakeNotePort implements NotePort {

### D.7.1 명명 규칙

| 대상 | 클래스명 |
|------|---------|
| Port (인터페이스) | `{Domain}Port`, `{Domain}EventPublisher`, `{Domain}Gateway` |
| gRPC Adapter | `{Domain}GrpcAdapter` |
| REST Adapter (외부 호출) | `{Domain}RestAdapter` |
| Kafka Producer Adapter | `{Domain}EventKafkaAdapter` |
| Kafka Consumer Listener | `{EventName}KafkaListener` |

| DTO | `{Operation}Dto` 또는 `{Domain}ResponseDto` |
| Mapper | `{Domain}Mapper` |

### D.7.2 패키지 별 어노테이션

```java
// 도메인 계층
@DomainService                              // 자체 stereotype (메타 어노테이션)

// 응용 계층
@UseCase                                    // = @Service + @Transactional(readOnly=true 등)

// 인프라 계층
@Component                                  // Adapter
@Mapper(componentModel = "spring")          // MapStruct
```

`@UseCase`, `@DomainService`는 자체 메타 어노테이션으로 정의:

```java
@Retention(RetentionPolicy.RUNTIME)
@Target(ElementType.TYPE)
@Service
@Transactional

```

## D.9 마이그레이션 가이드 (기존 코드 → 어댑터 패턴)

기존에 UseCase가 외부 SDK를 직접 호출하던 경우 단계별 리팩토링:

```
Phase 0 (Before): 
  UseCase ─→ NoteServiceBlockingStub (직접)

Phase 1: Port 추출
  - domain/port/outbound/NotePort 인터페이스 정의
  - infrastructure/adapter/outbound/note/NoteGrpcAdapter 구현
  - UseCase에서 stub 의존을 NotePort로 교체
  - DTO ↔ Domain Mapper 추출

Phase 2: 도메인 모델 정제
  - Adapter가 proto 모델을 도메인 record로 변환
  - 도메인 record에서 proto 필드명 모두 제거

Phase 3: 횡단 관심사 이동
  - @CircuitBreaker, @Retry 등을 Adapter로 이동
  - 로깅/추적을 Adapter에서 수행

Phase 4: ArchUnit 도입
  - 의존성 방향 강제
  - PR 차단
```
