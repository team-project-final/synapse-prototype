### 3.1 Backend — Spring Boot (JUnit 5 + Mockito)

```java
// 예시: SRS SM-2 알고리즘 단위 테스트
@ExtendWith(MockitoExtension.class)
class Sm2AlgorithmTest {

    private Sm2Algorithm sm2;

    @BeforeEach
    void setUp() {
        sm2 = new Sm2Algorithm();
    }

    @ParameterizedTest
    @CsvSource({
        "5, 2.5, 1, 6",    // quality=5, ease=2.5, rep=1 → interval=6
        "3, 2.5, 1, 6",    // quality=3, ease 유지
        "2, 2.5, 1, 1",    // quality<3, 리셋
    })
    void shouldCalculateNextInterval(int quality, double ease, int rep, int expectedInterval) {
        ReviewResult result = sm2.calculate(quality, ease, rep);
        assertThat(result.interval()).isEqualTo(expectedInterval);
    }

    @Test
    void shouldNotReduceEaseFactorBelowMinimum() {
        ReviewResult result = sm2.calculate(0, 1.3, 5);
        assertThat(result.easeFactor()).isGreaterThanOrEqualTo(1.3);
    }
}
```

**규칙:**
- 하나의 테스트 = 하나의 행위 검증
- Given-When-Then 구조
- Mock은 외부 의존성에만 사용 (DB, API, 메시지큐)
- 도메인 로직은 실제 객체로 테스트
- `@DisplayName` 한국어 사용 권장

### 6.1 SRS 알고리즘 검증

```java
@Nested
class Sm2AlgorithmVerification {

    @Test
    @DisplayName("SM-2: 연속 정답 시 간격이 지수적으로 증가한다")
    void intervalGrowsExponentially() {
        double ease = 2.5;
        int interval = 1;
        
        for (int rep = 0; rep < 10; rep++) {
            ReviewResult result = sm2.calculate(4, ease, rep); // quality=4 (Good)
            assertThat(result.interval()).isGreaterThan(interval);
            interval = result.interval();
            ease = result.easeFactor();
        }
    }

    @Test
    @DisplayName("SM-2: quality < 3 시 학습 리셋")
    void resetsOnFailure() {
        ReviewResult result = sm2.calculate(2, 2.5, 10); // 10회차에서 실패
        assertThat(result.interval()).isEqualTo(1);
        assertThat(result.repetition()).isEqualTo(0);
    }

    @Test
    @DisplayName("SM-2: EaseFactor 최솟값 1.3 보장")
    void easeFactorNeverBelowMinimum() {
        double ease = 1.3;
        for (int i = 0; i < 100; i++) {
            ReviewResult result = sm2.calculate(0, ease, i);
            ease = result.easeFactor();
            assertThat(ease).isGreaterThanOrEqualTo(1.3);
        }
    }
}
```

# LLM 응답을 결정론적으로 테스트하기 위한 Mock 전략
class MockLLMClient:
    """테스트용 LLM 클라이언트 — 고정된 응답 반환"""
    
    def __init__(self, responses: dict[str, str]):
        self.responses = responses
        self.call_log = []
    
    async def generate(self, prompt: str, **kwargs) -> str:
        self.call_log.append({"prompt": prompt, **kwargs})
        # 프롬프트 패턴에 따른 고정 응답
        for pattern, response in self.responses.items():
            if pattern in prompt:
                return response
        return self.responses.get("default", "")

@pytest.fixture

  integration-test:
    runs-on: ubuntu-latest
    services:
      postgres:
        image: pgvector/pgvector:pg16
        env:
          POSTGRES_DB: synapse_test
          POSTGRES_USER: test
          POSTGRES_PASSWORD: test
        ports: ['5432:5432']
    steps:
      - uses: actions/checkout@v4
      - name: Run Integration Tests
        run: ./gradlew integrationTest

  e2e-test:
    runs-on: ubuntu-latest
    needs: [unit-test, integration-test]
    steps:
      - uses: actions/checkout@v4
      - name: Start Services
        run: docker compose -f docker-compose.test.yml up -d
      - name: Run Playwright Tests
        run: npx playwright test
      - uses: actions/upload-artifact@v4
        if: failure()
        with:
          name: playwright-report
          path: playwright-report/
