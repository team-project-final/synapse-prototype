## 2. Epic & User Story

### Epic 1: 사용자 인증 및 계정 관리

#### US-1.1: 소셜 로그인으로 빠른 시작

> **As a** 개발자 김시냅스 (P1)
> **I want to** GitHub 계정으로 한 번에 로그인하고 싶다
> **So that** 별도 회원가입 절차 없이 바로 서비스를 사용할 수 있다

**Acceptance Criteria:**
- [ ] GitHub OAuth 버튼 클릭 시 GitHub 인증 화면으로 리다이렉트
- [ ] 인증 완료 시 자동 회원가입 + 로그인 처리
- [ ] 프로필 정보 (이름, 이메일, 아바타) 자동 연동
- [ ] 로그인 후 대시보드(SCR-DASHBOARD)로 이동
- [ ] 기존 회원이면 바로 로그인, 신규면 가입 처리

#### US-1.2: MFA 보안 설정

> **As a** 대학원생 이연구 (P2)
> **I want to** 내 계정에 2차 인증(MFA)을 설정하고 싶다
> **So that** 연구 노트 데이터를 안전하게 보호할 수 있다

**Acceptance Criteria:**
- [ ] 보안 설정 페이지에서 MFA 활성화 가능
- [ ] Google Authenticator 호환 QR 코드 표시
- [ ] 복구 코드 10개 발급 및 다운로드
- [ ] 다음 로그인부터 TOTP 코드 입력 요구
- [ ] 복구 코드로 MFA 해제 가능

### Epic 2: 노트 관리 (PKM)

#### US-2.1: 마크다운 노트 작성

> **As a** 개발자 김시냅스 (P1)
> **I want to** 마크다운으로 기술 블로그 내용을 정리하고 싶다
> **So that** 익숙한 문법으로 빠르게 지식을 기록할 수 있다

**Acceptance Criteria:**
- [ ] 에디터에서 마크다운 실시간 프리뷰 지원
- [ ] 코드 블록 (```lang```) 구문 하이라이팅
- [ ] 이미지 드래그 앤 드롭 업로드
- [ ] 3초 디바운스 자동 저장
- [ ] Ctrl+S 수동 저장도 가능

#### US-2.2: 위키링크로 노트 연결

> **As a** 대학원생 이연구 (P2)
> **I want to** [[개념명]] 형식으로 노트 간 링크를 만들고 싶다
> **So that** 논문에서 읽은 개념들의 관계를 구조화할 수 있다

**Acceptance Criteria:**
- [ ] `[[` 입력 시 기존 노트 자동완성 드롭다운 표시
- [ ] 존재하는 노트 선택 시 링크 생성
- [ ] 존재하지 않는 제목 입력 시 "새 노트 생성" 옵션 표시
- [ ] 링크된 노트에서 백링크 섹션에 현재 노트 표시
- [ ] 링크 클릭 시 해당 노트로 이동

#### US-2.3: 전문 검색으로 과거 노트 찾기

> **As a** 대학원생 이연구 (P2)
> **I want to** 키워드로 과거에 정리한 노트를 빠르게 찾고 싶다
> **So that** 기존 정리 내용을 쉽게 참조하고 연결할 수 있다

**Acceptance Criteria:**
- [ ] 검색창에서 실시간 검색 결과 표시 (디바운스 300ms)
- [ ] 한국어 형태소 분석 적용 (예: "학습했다" → "학습" 매칭)
- [ ] 검색 결과에 매칭 부분 하이라이팅
- [ ] 제목/본문/태그 통합 검색
- [ ] 최근 검색어 기록

### Epic 3: 카드 학습 (SRS)

#### US-3.1: 노트에서 수동 카드 생성

> **As a** 자격증 준비자 박합격 (P3)
> **I want to** 정리한 내용에서 중요 부분을 선택하여 플래시카드를 만들고 싶다
> **So that** 핵심 개념을 반복 복습할 수 있다

**Acceptance Criteria:**
- [ ] 노트 텍스트 드래그 선택 후 "카드 만들기" 버튼 표시
- [ ] 선택 텍스트가 카드 뒷면에 자동 입력
- [ ] 앞면(질문) 직접 입력 또는 AI 제안
- [ ] 생성 시 원본 노트 링크 유지
- [ ] 저장할 덱 선택 가능

#### US-3.2: 일일 복습 세션

> **As a** 자격증 준비자 박합격 (P3)
> **I want to** 매일 복습 예정 카드를 순서대로 학습하고 싶다
> **So that** SM-2 알고리즘에 따라 효율적으로 암기할 수 있다

**Acceptance Criteria:**
- [ ] 앱 실행 시 오늘 복습 카드 수 표시
- [ ] "복습 시작" 버튼으로 세션 진입
- [ ] 카드 앞면 표시 → 탭으로 뒷면 공개
- [ ] 1(Again) ~ 4(Easy) 평가 버튼 표시
- [ ] 평가 후 다음 카드 자동 전환
- [ ] 세션 종료 시 결과 요약 (정답률, 소요 시간)

#### US-3.3: 복습 통계 확인

> **As a** 자격증 준비자 박합격 (P3)
> **I want to** 내 학습 진행 상황과 약점을 파악하고 싶다
> **So that** 부족한 부분에 더 집중할 수 있다

**Acceptance Criteria:**
- [ ] 일별 복습 수 히트맵 (GitHub 잔디 스타일)
- [ ] 덱별 정답률 막대 그래프
- [ ] 주간/월간 복습 추이 차트
- [ ] "약한 카드" 목록 (정답률 하위 20%)
- [ ] 예측 복습량 (향후 7일)

### Epic 4: AI 기능

#### US-4.1: AI 카드 자동 생성

> **As a** 대학원생 이연구 (P2)
> **I want to** 논문 노트에서 AI가 자동으로 복습 카드를 만들어 주길 원한다
> **So that** 카드 제작 시간을 줄이고 핵심 개념 학습에 집중할 수 있다

**Acceptance Criteria:**
- [ ] 노트 편집 화면에서 "AI 카드 생성" 버튼
- [ ] 생성 수량 선택 (1~20장)
- [ ] 생성 중 로딩 표시 (스트리밍)
- [ ] 생성된 카드 미리보기 목록 표시
- [ ] 개별 카드 편집/삭제/승인 가능
- [ ] "모두 저장" 시 선택된 카드 덱에 추가

#### US-4.2: 시맨틱 검색

> **As a** 개발자 김시냅스 (P1)
> **I want to** 정확한 키워드를 몰라도 의미적으로 관련된 노트를 찾고 싶다
> **So that** 다양한 표현으로 기록한 노트도 빠짐없이 찾을 수 있다

**Acceptance Criteria:**
- [ ] 검색 모드 토글: 키워드 / 시맨틱 / 하이브리드
- [ ] 시맨틱 검색 시 의미 유사 노트 상위 10개 반환
- [ ] 유사도 점수 표시 (0~100%)
- [ ] 검색 결과에서 관련 구절 하이라이팅
- [ ] 검색 속도 500ms 이내

## 3. 사용자 여정 맵 (User Journey)

### P3. 박합격 — 첫 사용부터 구독까지

```
[1] 앱 설치 → [2] Google 로그인 → [3] 온보딩 투어 
    → [4] 샘플 노트 확인 → [5] AWS 학습 노트 작성 
    → [6] AI 카드 생성 (무료 할당량) → [7] 첫 복습 세션 
    → [8] 매일 복습 루틴 형성 → [9] 무료 할당량 소진 
    → [10] Pro 구독 → [11] 시험 합격!
```

### 핵심 전환 포인트

| 단계 | 전환 트리거 | 이탈 방지 |
|------|------------|-----------|
| 3→4 | 온보딩 완료 보상 (샘플 제공) | 3분 이내 첫 가치 경험 |
| 6→7 | AI 생성 카드 품질 만족 | 카드 편집 기능으로 보완 |
| 9→10 | 할당량 소진 알림 + 학습 통계 제시 | "지금까지 150장 복습, Pro로 계속하세요" |
