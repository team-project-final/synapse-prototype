# 1. 개요

### 1.1 문서 목적

본 문서는 Synapse 프로젝트에서 사용하는 **모든 기술 스택**을 백과사전 형식으로 정의한다. 각 기술의 선택 근거, 기술적 이점, 설정 가이드, 트러블슈팅을 포함하여 신규 팀원 온보딩 및 기술 의사결정의 단일 참고 문서로 활용한다.

**본 문서가 다루는 내용:**
- 모든 기술 선택의 배경과 대안 비교
- 각 기술의 Synapse 프로젝트 내 구체적 역할
- 핵심 설정 항목 및 예시 코드
- 운영 중 자주 발생하는 문제와 해결책
- 공식 문서 및 참고 자료 링크

### 1.3 기술 선택 기준

Synapse 프로젝트의 모든 기술 선택은 아래 6가지 기준을 가중치 순으로 평가하여 결정하였다.

| 기준 | 가중치 | 설명 |
|------|--------|------|
| **성숙도 (Maturity)** | 25% | GA 릴리스 여부, 프로덕션 검증 사례, CVE 대응 이력 |
| **커뮤니티 (Community)** | 20% | GitHub Stars, Stack Overflow 답변 수, 활성 유지보수 |
| **성능 (Performance)** | 20% | 벤치마크 수치, 지연시간, 처리량 목표 달성 가능성 |
| **확장성 (Scalability)** | 15% | 수평 확장 지원, 파티셔닝/샤딩 전략, 클러스터 모드 |
| **학습 곡선 (Learning Curve)** | 10% | 팀 기존 역량, 공식 문서 품질, 온보딩 시간 |
| **팀 역량 (Team Capability)** | 10% | 팀 내 경험자 보유 여부, 상용 지원 가용성 |
